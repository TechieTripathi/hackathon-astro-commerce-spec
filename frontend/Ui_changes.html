Application Core Changes & Roadmap

1. Missing Customer Pages & Routes

  • /checkout/payment (Dedicated Payment Loading/Interim Page): Before routing directly to a success state—when the Razorpay or Cashfree SDK triggers a callback—a loading state and web verification screen are required so the UI doesn't crash if the user abruptly closes the window.
  • /server-error (500) and /404 (Not Found) Pages: While error handling is outlined in the state contract, explicit and dedicated fallback pages for these statuses are missing from your route map.
  • /track-order/:orderId (Live Tracking UI View): You already have the order detail view (/account/orders/:orderId), but the customer experience lacks an interactive, step-by-step generic vertical visual tracker or map-style timeline.
  • Support/Help Center View (/support or /faq): Where will a customer go to resolve order or refund queries? An interactive FAQ component or a support ticket log UI is currently missing.

2. Missing Admin Pages & Routes

  • /admin/banners or /admin/cms (Content Management System): The customer application includes sections like Featured categories, Featured products, and Recommendations blocks, but there is no interface for an admin to update them. A dedicated layout manager page is required to upload banners and toggle homepage sections.
  • /admin/users/:userId (User Activity & Detail View): The user list view (/admin/users) exists, but a detailed view to open a specific user profile and monitor their order history, active items in cart, and saved addresses is missing (this is vital for Customer Support Agents).
  • /admin/reports (Analytics & Export Page): The dashboard only displays card metrics. A frontend control panel to download or export sales data, tax reports, or low-stock items into CSV/Excel formats is missing.

3. Frontend State Contract Extensions (Core FE Logic)

The state contract must explicitly address and handle these specific rules and UI states:

Form Layer & Validation Extensions

  • Dirty Form Guard (Unsaved Changes Warning): If an admin is editing a product or a customer is filling out the checkout form and accidentally hits the back button, the browser/router should intercept it with an alert ("Are you sure you want to leave? Unsaved changes will be lost").
  • Dynamic Attribute Matrix Generator: When an admin selects attributes like carat or metal in the admin panel, the frontend should dynamically generate all possible combinations (SKUs) into a clean data table where the admin can easily populate the price and stock for each variant.

Optimization & UX States

  • Skeleton Loaders Strategy: Avoid generic spinners across data-fetching pages. Instead, implement layout-specific Shimmer/Skeleton components (e.g., card skeletons for the product listing grid).
  • Network Status Toast (Offline/Online Sync): If a user attempts to place an order and their internet drops, the frontend must immediately display a status overlay to prevent duplicate charges or API calls.
  • Infinite Scroll vs. Pagination State UI: If the product list utilizes infinite scrolling, the frontend must sync the page state inside the URL query parameters so that clicking the back button preserves the exact scroll position (Scroll Restoration).

Security & Storage Contract

  • Token Expiry Warning (Session Timeout Modal): If the JWT access token is nearing its expiration (especially critical within the Admin app), display a modal warning that gives the user a chance to explicitly extend their session.
